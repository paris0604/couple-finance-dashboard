# 🏡 우리집 자산 대시보드 (Couple Finance Dashboard)

신혼부부 **합산 가계(공동 풀)** 의 월 현금흐름·**저축률**·순자산을 한 화면에서.
[PRD v0.2](../PRD_couple_finance_dashboard.md) 구현체 · Streamlit.

> **핵심 지표** — 저축률 = (예적금 + 투자납입) ÷ 합산수입
> **성공 기준** — 매월 결산 5분 이내, 두 사람이 *같은 숫자* 로 의사결정 (PRD §1)

---

## 빠른 시작 (키 없이 바로 실행)

```bash
cd couple-finance-dashboard
pip install -r requirements.txt
streamlit run app.py
```

`secrets.toml` 이 없으면 **샘플 데이터 모드**로 즉시 동작합니다(3개월치 예시 가계/투자 데이터 포함). 빠른입력으로 추가하는 거래는 `sample_data/가계부.csv` 에 쌓입니다.

샘플을 다시 생성하려면: `python sample_data/_generate.py`

---

## Google Sheets 연동 (실데이터)

1. Google Cloud에서 **서비스 계정** 생성 → JSON 키 발급
2. 대상 스프레드시트를 서비스 계정 이메일에 **편집자**로 공유
3. `.streamlit/secrets.toml.example` → `secrets.toml` 로 복사 후 값 입력
4. 앱 재실행 → 사이드바가 🟢 **Google Sheets 연결됨** 으로 바뀝니다

- `가계부` 시트가 없으면 표준 헤더로 자동 생성합니다.
- `보유현황`(투자) 시트는 **읽기 전용** — 코드가 절대 쓰지 않습니다 (PRD §3).

## 3개 뷰 (PRD §9 MVP)

| 뷰 | 내용 |
|---|---|
| 🏠 **공통 요약** | 저축률(히어로) · 현금흐름 워터폴 · 자금흐름(공통/지영/승화) · **자산설계 인사이트** · 월별 추이 |
| 📒 **가계부** | 빠른입력(규칙기반 자연어) · 고정/변동/용돈 · 카테고리별 지출 · 용돈 현황 · 거래내역 |
| 📈 **투자** | 거래내역 → **순투입원금**(시세 불필요) · 종목·계좌별 집계 |

### 자산설계 인사이트 (20년차 설계사 관점)
공통 요약 하단에 단순 집계를 넘는 진단 카드 제공:
- 🎯 **저축률 등급 + 목표 역산** — 50% 목표까지 얼마를 더 모으면 되는지
- 🟢/🔴 **비상예비자금** — 월 지출 × 3~6개월 대비 현금 버퍼 충분성
- **고정비 비중** — 수입의 30~40% 건강선 진단
- 🚀 **연 환산 자산형성 속도**

기준값은 `core/config.py` 의 `SAVINGS_RATE_TARGET`, `EMERGENCY_FUND_MONTHS`, `ALLOWANCE_PER_PERSON` 한 곳에서 조정합니다.

## 빠른입력 예시 (규칙 기반, 비용 0)

```
어제 마트 3.2만            → 어제 · 식비-장보기 · 32,000
점심 김밥 5천 외식          → 오늘 · 식비-외식 · 5,000
6/1 월세 80만 주거          → 06-01 · 주거 · 800,000
지영 용돈 40만              → 용돈 · 지출구분=지영 · 400,000
승화 급여 360만 수입         → 수입 · 승화 급여 · 3,600,000
```

변환 결과는 저장 전 폼에서 확인·수정할 수 있습니다. (Claude API 고도화는 PRD §13 2차 검토)

## 구조

```
couple-finance-dashboard/
├── app.py                 # Streamlit 진입점 (3개 뷰)
├── core/
│   ├── config.py          # 카테고리·스키마·설계 기준값 (SSOT)
│   ├── data_loader.py     # Google Sheets ↔ 샘플 CSV 자동 전환
│   ├── finance.py         # 저축률·순투입원금·순자산·인사이트 계산
│   └── nlp_input.py       # 규칙 기반 자연어 정규화
├── sample_data/           # 키 없이 실행되는 예시 데이터 + 생성기
└── .streamlit/            # 화이트톤 테마 + secrets 템플릿
```

모든 컬럼은 **헤더(이름) 기준** 접근 — 시트 열 순서나 `schema_version` 변동에 안전 (PRD §10).
